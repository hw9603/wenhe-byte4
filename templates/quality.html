Quality: About This Data Set

The overal quality of this data set is good. It contains some blank values and some noises. Apart from that, the data is clean and well structured.

Is the data complete?

The table for activities is quite complete. It contains only a few rows of null value for the predicted activities column. As can be observed from the following graph, the number of null rows only accounts for a very small amount of data. However we can see that the data collected at some time period is not sufficient (6-11 significantly less). This might be caused by not asiding enough time for the data collection process or the mobile phone blocking issue.

Is the data coherent?

As can be observed from the above graph, the distribution of activities over different hours of a day quite makes sense. Since the data is recorded in UTC timezone, we can see that the user tends to be more stationary at night and more active during the daytime. This matches with the common sense.

Is the data correct?

Since the activity data belongs to my own daily activity, I know what was happening in every day during the data collection process. In order to verify the data, I plot the date-activity graph so that I can see what activities I did for each day. As can be observed in the below graph, there are some records of automotive activities in date 13, 14, 15, 16. This was exactly what happened to me in those dates. I rented a car in those four days and drove for sometime, while I did not drive in other days. For date 7, there were records of automotive activities because I took the bus to the shopping mall. In the other days, I just walked to schools and stayed stationary for studying, sleeping and resting.
